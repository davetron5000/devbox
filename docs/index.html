Table of Contents

100 pages, 15 chapters. Here they are:

  1. Your Dev Environment Sucks. What problem are we solving? You can feel it in your bones.

  2. A Dev Environment Defined. There's some VC that thinks dev experience is making a better version of New Relic and that's not what's happening. Dev environment is where you write your code.

  3. The Four Types of Dev Environments. Includes a Gartner quadrant to prove, without a doubt, that this book will help you build the best dev environment. The other three types are terrible.

  4. What is Docker and How Does It Work?. This chapter is worth the price alone, because it provides a Docker-to-English dictionary!

  5. Managing the Operating System With Base Images. Learn the convoluted process by which you try to determine if your operating system vendor wants you downloading images from the Internet.

  6. Installing Software in the Dev Environment. Aaaaaand, this is where I explain—or at least try to—how to safely trust piping curl into bash for the purposes of installing software on a Linux box.

  7. Publishing Ports and Mounting Volumes to Do Development. Seems simply enough to connect your Docker stuff to the outside world, but it's not. At least not until you read this chapter.

  8. Running Databases and Caches with One Line of Code. Say good bye to figuring out why some old version of Postgres no longer works on MacOs Bakersfield (or whatever).

  9. What is a Good User Experience for Automation Scripts?. Before this chapter, you will have learned how awesome Docker actually us, but the command line part will still suck. After this chapter, you will know what proper command line developer experience is.

  10. The Arcane World of the UNIX Command Line. In which I explain exit codes, output streams, variable expansion, and quotation marks.

  11. Writing One Nice Script. The most bash code you will ever read in a book that isn't about bash.

  12. Build the Remaining Scripts. Did you know you can apply normal software engineering principles to bash scripts? You will after this chapter.

  13. Sustainably Maintaining The Dev Environment. Yes, you can set it up, but you also need to keep it going.

  14. There’s No Such Thing as a Free Lunch. Nothing is easy in this world. This part includes an entire section on how strange or weird Microsoft's Dev Containers thing is.

  15. Colophon or, The Chapter About Fonts. Everyone loves fonts.
